## Getting Started with the API

To begin using the EVO API, you'll need an API key and basic understanding of REST principles.

<Steps>
  <Step title="Generate API Key" icon="key">
    In your EVO dashboard, navigate to Settings > API Keys and create a new key with appropriate permissions.

    ```bash
curl -X POST https://api.evo-docs.com/keys \
  -H "Authorization: Bearer $EVO_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"CI/CD Integration","permissions":["read","write"]}'
    ```
  </Step>
  <Step title="Test Connection" icon="check-circle">
    Verify your API key works by making a simple request to retrieve your user information.
  </Step>
  <Step title="Explore Endpoints" icon="search">
    Review the API documentation to understand available endpoints and their parameters.
  </Step>
</Steps>

## Core API Operations

The EVO API supports all major CRUD operations for managing documentation spaces, content, and users.

<Tabs>
  <Tab title="Spaces" icon="folder">
    Manage documentation spaces programmatically.

    ```javascript
// Create a new space
const response = await fetch('https://api.evo-docs.com/spaces', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${API_KEY}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    name: 'API Documentation',
    description: 'Comprehensive API reference',
    visibility: 'public'
  })
});
    ```
  </Tab>
  <Tab title="Documents" icon="file-text">
    Create, update, and manage individual documents.

    ```python
import requests

headers = {'Authorization': f'Bearer {api_key}'}
data = {
    'title': 'Getting Started',
    'content': '# Welcome to EVO\n\nThis is your first document.',
    'space_id': space_id
}

response = requests.post('https://api.evo-docs.com/documents', 
                        headers=headers, json=data)
    ```
  </Tab>
  <Tab title="Users" icon="users">
    Manage user accounts and permissions.

    ```bash
# List space members
curl -H "Authorization: Bearer $API_KEY" \
     https://api.evo-docs.com/spaces/$SPACE_ID/members
    ```
  </Tab>
</Tabs>

## Official SDKs

We provide official SDKs for popular programming languages to simplify integration.

<CodeGroup tabs="JavaScript,Python,Go">
  ```javascript
// Install: npm install evo-docs-sdk
const { EVODocs } = require('evo-docs-sdk');

const client = new EVODocs({ apiKey: 'your-key' });
const spaces = await client.spaces.list();
  ```
  ```python
# Install: pip install evo-docs-sdk
from evo_docs import EVODocs

client = EVODocs(api_key='your-key')
spaces = client.spaces.list()
  ```
  ```go
// Install: go get github.com/evo-docs/sdk-go
package main

import (
    "github.com/evo-docs/sdk-go"
)

func main() {
    client := evo.NewClient("your-api-key")
    spaces, _ := client.ListSpaces()
}
  ```
</CodeGroup>

## Error Handling and Best Practices

Build robust integrations by implementing proper error handling and following API best practices.

- **Rate Limiting**: Respect API rate limits (100 requests per minute for most endpoints).
- **Error Responses**: Handle HTTP error codes appropriately (400 for bad requests, 401 for authentication issues).
- **Idempotency**: Use idempotent operations where possible to avoid duplicate actions.
- **Pagination**: Implement pagination for list endpoints to handle large result sets.
- **Authentication**: Rotate API keys regularly and use the minimum required permissions.

By leveraging EVO's comprehensive API, you can create powerful integrations that enhance your team's documentation workflow and ensure your documentation stays current and accurate.
